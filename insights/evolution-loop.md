# 数据驱动的自进化机制

> 核心命题：**确定性 Pipeline + 可进化的决策参数**。流水线的阶段顺序、接口契约、执行机制不变，但每个阶段内部的决策参数由外部数据反馈持续调优。目标是用尽量低的成本制作出更多受欢迎的内容。

## 为什么进化机制是核心差异化

行业 0.16% 的爆款命中率（见 [reflections.md](reflections.md) 第 1 节）说明**"知道该做什么"比"做得好看"值钱得多**。FreeToon 40% 的爆款率不是因为画质更好，而是因为选题更准、节奏更好——这些是可积累的经验。

当前所有工具（包括 ArcReel）都在解决"怎么做"（how to produce），没有一个在解决"怎么做得越来越好"（how to improve over time）。进化机制就是回答后者的方式。

---

## 核心循环

```
制作 → 投放 → 收集外部数据 → 归因到具体环节 → 调整决策参数 → 再制作
  ↑                                                           │
  └───────────────────────────────────────────────────────────┘
```

这不是一次性优化，而是一个**持续运转的闭环**。每一轮循环都让系统对"什么样的内容受欢迎"有更精确的认知，并把这个认知固化为决策参数。

---

## 什么不变，什么要进化

| 阶段 | 确定性的部分（不变） | 可进化的决策参数 | 进化数据来源 |
|------|-------------------|----------------|------------|
| **剧本理解** | 输入剧本 → 输出 StoryBlueprint | 什么样的故事结构完播率高？集数、节奏怎么切？开头几秒怎么设计钩子？ | 完播率曲线、弃剧集数分布 |
| **角色设计** | 输入角色描述 → 输出视觉参考 | 什么画风在目标平台转化率高？角色设计的哪些特征（大眼、高饱和度、夸张比例）更吸引点击？ | 封面点击率、角色相关弹幕/评论情感 |
| **分镜** | 输入场景 → 输出 ShotSpec | 哪种镜头策略组合留存率高？节奏快慢和平台偏好的关系？ | 分段完播率（精确到秒级）、快进/回看热点 |
| **图片生成** | 输入 prompt → 输出图片 | 哪个 Provider/Model 在当前风格下性价比最优？prompt 怎么写效果最好？ | 质检通过率、重试次数、单镜头成本 |
| **视频生成** | 输入图片+运镜 → 输出视频 | 同上，加上运镜参数与观感的关系 | 同上 |
| **合成** | 输入视频片段 → 输出成片 | 转场类型和时长对完播率的影响？BGM 风格偏好？ | 分段完播率、用户反馈 |

## 外部数据体系

### 需要采集的数据

```
一级指标（平台直接提供）：
├── 播放量、完播率、点赞率、评论数、收藏数、分享数
├── 封面点击率（CTR）
├── 续集观看率（第 N 集 → 第 N+1 集的转化率）
└── 收入数据（如果有投流/付费模式）

二级指标（需要加工）：
├── 分段完播率曲线（第几秒开始流失）
├── 弃剧集数分布（观众在第几集弃剧）
├── 角色偏好信号（弹幕/评论中角色提及频次和情感）
└── 风格偏好信号（同题材不同画风的效果对比）

三级指标（跨项目积累）：
├── 题材-平台匹配度（什么题材在什么平台表现好）
├── 时间窗口效应（发布时间对数据的影响）
└── 竞品数据（同期同题材作品的表现）
```

## 归因 — 最难也最关键的环节

一部剧数据差，可能是剧本问题、画风问题、节奏问题、分发时段问题，甚至是封面问题。**确定性 Pipeline 的优势在于每个参数都可以独立控制和度量**——这是 Agent 自由发挥的黑盒做不到的。

### 三种归因策略

**策略 1：A/B 实验**

```
控制变量：同一剧本、同一角色、同一模型
实验变量：不同的分镜策略（策略 A vs 策略 B）
→ 生成两版成片 → 同时投放 → 对比完播率
→ 策略 A 的完播率比策略 B 高 12%
→ 策略 A 在该场景类型下的权重 +0.12
```

可做 A/B 的维度：分镜策略、画风参数、节奏切分、开头钩子设计、封面选择。

**策略 2：回归分析**（无需实验，回顾历史数据）

当积累了足够多的项目后，分析特征（题材、画风参数、节奏特征、策略分布）与结果（完播率、续集转化率）的相关性，识别哪些特征组合与高完播率强相关。

**策略 3：秒级归因**（最有价值）

如果能获取分段完播率曲线（很多平台提供），可以把流失点精确映射到具体镜头：

```
完播率曲线：0s:100% → 3s:72% → 8s:65% → 15s:58% → 30s:45%

映射到内部结构：
  0-3s  → 第 1-2 个镜头 → 开头钩子不够强
  8-15s → 第 5-8 个镜头 → 叙事节奏拖沓
  → 精确定位到具体的镜头策略和节奏参数
```

## short-maker 现有设计中的进化种子

当前代码中有三个机制天然适合接入进化循环，架构已经对了，缺的是数据反馈。

### 1. ImportanceScore — 资源分配权重

当前是固定值（`climax=2.0, hook=1.5` 等），决定每个镜头的重试次数和质量阈值。这些权重未来应由投放数据驱动——比如数据发现前 3 秒流失率高，说明 `open_hook` 的重要性被低估了，应上调权重，让开头镜头获得更多生成资源。

### 2. Strategy Engine — 策略效果积累

当前 10 种镜头策略只有 tag 匹配打分。未来每个策略应积累使用次数、对应片段的平均完播率、最佳/最差适用场景。`Matcher` 打分时不仅看 tag 匹配，还看历史表现。

### 3. QualityChecker — 校准"好"的定义

当前 5 个评分维度的权重是人工设定的。未来应验证：`quality_score=85` 的镜头投放后效果一定比 `score=70` 的好吗？通过回归分析校准权重，让质检评分更准确地反映"观众觉得好"而非"技术上好"。

## 进化的三个阶段

### 阶段 1：手动记录 + 人工分析

投放后手动录入数据，人工分析，手动调整参数。价值在于**开始积累数据，验证归因逻辑是否成立**。

### 阶段 2：半自动归因 + 参数建议

对接平台数据 API 自动拉取投放数据，系统计算相关性并生成参数调整建议，人工审核后应用。

### 阶段 3：全自动闭环

自动采集 → 自动归因 → 自动调参 → 自动 A/B → 自动择优。设安全边界防止系统发散。

## 与 ArcReel 等现有工具的本质区别

```
现有工具（ArcReel、巨日禄、火山剧创...）：
  定位 → 制作工具
  优化对象 → 单次制作的效率和体验
  反馈循环 → 无（制作完成即结束）
  越用越好？→ 不会，第 100 部和第 1 部用的是同样的参数

short-maker 的方向：
  定位 → 会学习的制作系统
  优化对象 → 跨项目积累的决策参数
  反馈循环 → 制作 → 投放 → 数据 → 归因 → 调参 → 再制作
  越用越好？→ 第 100 部的参数是从前 99 部的数据中学到的
```

工具的价值上限是"用起来顺手"，系统的价值上限是"用得越多越准"。
