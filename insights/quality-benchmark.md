# AI 漫剧质量基准

从调研中建立的质量评估体系，用于：
1. **横向对比**不同 AI 漫剧工具的产出质量
2. 作为**自建 Agent 系统的质量目标**和标尺

---

## 基准作品：《西游，错把玉帝当亲爹》

### 基本信息

| 项目 | 信息 |
|------|------|
| 制作方 | 广州星火深智动漫有限公司（星火动漫） |
| 制作工具 | FreeToon（星火动漫自研） |
| 平台 | 红果短剧 |
| 播放量 | **2.3 亿+**（上线 35 小时破亿） |
| 热度 | 红果平台首部热度破 6000 万的漫剧 |
| 收藏量 | 200 万+ |
| 制作模式 | 211 模式（20 天、10 万元算力成本、10 人团队） |
| 风格 | 2D 国漫风格 |
| 上线时间 | 2026 年 2 月 15 日（春节档） |

### 为什么选它做基准

1. **市场验证最充分** — 2.3 亿播放量，是 AI 漫剧行业目前播放量最高的作品之一
2. **纯 AI 工具生产** — 全流程使用 FreeToon 制作，代表 AI 漫剧工具的产出上限
3. **商业成功** — 上线即爆款，证明 AI 制作的漫剧可以达到商业发行标准
4. **可复制性** — 星火动漫用同一套工具和方法论制作了 33 部作品，13 部播放破千万，说明不是单次运气
5. **行业标杆** — 春节档新增播放总量第一的 AI 短剧厂牌

## 评估维度与评分体系

总共 **7 个评估维度**，每个维度 **0-10 分**，加权汇总为综合评分。

### 评分标准通用说明

| 分数段 | 含义 |
|--------|------|
| **0** | 完全不可用 |
| **1-3** | 严重缺陷，影响观看体验 |
| **4-5** | 勉强可用，明显不足 |
| **6-7** | 合格，能上平台但有改进空间 |
| **8-9** | 优秀，接近商业发行标准 |
| **10** | 标杆水准，当前 AI 工具的产出上限 |

---

## 维度 1：角色视觉一致性

> 同一角色跨镜头、跨集的面部、服装、身形、配色的一致程度

**权重：20%**（漫剧最核心的技术挑战）

### 评分标准

| 分数 | 水平描述 |
|------|---------|
| **0** | 同一角色每次出场像不同的人 |
| **3** | 能看出是同一角色但细节频繁变化（发色/服装/五官不稳定） |
| **5** | 大部分镜头一致，但偶尔出现崩坏（如侧脸/远景时不稳定） |
| **7** | 跨镜头高度一致，仅在极端角度或运动中偶有偏差 |
| **10** | 全集所有镜头角色完全一致，不同集数之间无法分辨差异 |

### 基准参考：《西游玉帝》 — **8 分**

表现：
- 主角（小玉帝）跨集形象高度一致，面部特征、发型、服装配色在不同场景中保持稳定
- 配角（玉帝、各路神仙）各有辨识度，整体一致性良好
- 偶尔在动态镜头或远景中出现细微偏差，但不影响观看
- 采用 FreeToon 的"多状态资产库"技术，资产继承贯穿全集

参考截图：[`screenshots/character-consistency-*.png`](screenshots/)

## 维度 2：画面质量与美术风格

> 画面精细度、光影效果、色彩搭配、构图水平、整体美术水准

**权重：20%**

### 评分标准

| 分数 | 水平描述 |
|------|---------|
| **0** | 涂鸦级画质，无法辨认内容 |
| **3** | 画面模糊、色彩单调、构图混乱 |
| **5** | 画面清晰但风格粗糙，缺乏美术设计感 |
| **7** | 画面精美，有统一的美术风格，光影和色彩搭配合理 |
| **10** | 媲美专业动画工作室水平，光影层次丰富、构图讲究、色彩表现力强 |

### 基准参考：《西游玉帝》 — **8.5 分**

表现：
- 国漫风格统一且精美，色彩饱满、场景氛围感强
- 天庭宫殿、云海、战斗场景等背景精致，光影层次分明
- 人物造型设计有辨识度，符合国漫审美
- 整体美术水准显著高于"巨日禄风"（行业早期 AI 漫剧的粗糙画风），证明了 AI 漫剧可以达到较高美术标准

参考截图：[`screenshots/art-quality-*.png`](screenshots/)

## 维度 3：动态与运镜

> 动作流畅度、镜头运动自然度、打斗/特效表现力、画面动态丰富度

**权重：15%**

### 评分标准

| 分数 | 水平描述 |
|------|---------|
| **0** | 纯静态图片，无任何动态效果 |
| **3** | 有简单的平移/缩放，但动作僵硬、不自然 |
| **5** | 基本的动态效果可用（人物动作、镜头推拉），但打斗/复杂场景表现差 |
| **7** | 动作流畅，镜头运动有节奏感，打斗场景有冲击力 |
| **10** | 动态效果接近传统动画水准，运镜语言丰富、动作编排精细 |

### 基准参考：《西游玉帝》 — **7.5 分**

表现：
- 日常叙事镜头动态自然，角色表情和肢体动作有变化
- 打斗/法术场景有较好的视觉冲击力，特效表现力强
- 镜头运用多样（推拉、跟随、特写切换），有影视感
- 仍然能看出 AI 生成的动态痕迹（如某些镜头的运动不够丝滑），但整体已远超静态漫画的水平

参考截图：[`screenshots/dynamics-*.png`](screenshots/)

## 维度 4：叙事结构与节奏

> 故事钩子设计、剧情节奏控制、情感起伏设计、完播率表现

**权重：20%**（漫剧商业成功的核心驱动因素）

### 评分标准

| 分数 | 水平描述 |
|------|---------|
| **0** | 无叙事可言，画面堆砌 |
| **3** | 有基本故事线但节奏拖沓，缺乏钩子 |
| **5** | 故事完整但平淡，缺少反转和高潮 |
| **7** | 节奏明快、钩子有效、每集有明确的情感高潮和悬念 |
| **10** | 精准的短剧叙事节奏，每 15-30 秒一个钩子，让人无法停止观看 |

### 基准参考：《西游玉帝》 — **9 分**

表现：
- 标题即钩子 —"错把玉帝当亲爹"制造强烈好奇心
- 开篇即冲突 — 迅速建立核心矛盾，不浪费时间铺垫
- 短剧节奏精准 — 每集结尾悬念感强，拉动续集观看（上线 35 小时破亿说明完播率/续播率极高）
- 上线第二天即启动第二季，说明用户追剧意愿强
- 情感起伏设计到位：搞笑反差（把玉帝当爹的荒诞感）+ 热血（主角成长/战斗）

> 注：叙事质量更多取决于剧本本身而非工具，但工具的分镜拆解和节奏控制能力会影响最终呈现。

## 维度 5：音频质量

> 配音自然度、音色与角色匹配度、音效设计、背景音乐搭配

**权重：10%**

### 评分标准

| 分数 | 水平描述 |
|------|---------|
| **0** | 无音频 |
| **3** | 有配音但机械感强，音色与角色严重不匹配 |
| **5** | AI 配音可接受但缺乏情感，音效/配乐基础 |
| **7** | 配音自然、音色匹配，音效和配乐增强了观看体验 |
| **10** | 配音水准接近专业声优，音效精细、配乐量身定做 |

### 基准参考：《西游玉帝》 — **7 分**

表现：
- FreeToon 集成腾讯云 AI 配音，多角色零样本即时配音
- 配音整体自然度尚可，能区分不同角色
- 音色匹配度良好，但与专业声优仍有差距
- 背景音乐和音效基本到位

> 注：星火动漫的配音方案已优于巨日禄（巨日禄不做配音，完全依赖第三方），但距离"以假乱真"仍有差距。

## 维度 6：技术完成度

> 分辨率、AI 伪影/崩坏率、字幕质量、画面稳定性、技术瑕疵密度

**权重：10%**

### 评分标准

| 分数 | 水平描述 |
|------|---------|
| **0** | 画面模糊不可看，严重伪影 |
| **3** | 频繁出现 AI 崩坏（多指、面部扭曲、物理不合理），影响观看 |
| **5** | 偶尔出现 AI 伪影但可忽略，分辨率达标 |
| **7** | 很少出现 AI 伪影，分辨率高，字幕清晰 |
| **10** | 零可见 AI 伪影，高分辨率，全技术环节无可挑剔 |

### 基准参考：《西游玉帝》 — **7.5 分**

表现：
- 画质经过腾讯云媒体处理转高清，最终呈现分辨率达标
- AI 伪影/崩坏率较低，大部分画面干净
- 字幕清晰，支持多语种（FreeToon 支持 30+ 语种字幕）
- 偶尔可见 AI 生成痕迹（如某些画面的手部细节、背景物件逻辑），但不影响整体观看

## 维度 7：商业可用性

> 成品能否直接上平台分发、是否需要大量人工后期、商业发行的就绪程度

**权重：5%**

### 评分标准

| 分数 | 水平描述 |
|------|---------|
| **0** | 完全不可发布的半成品 |
| **3** | 需要大量人工后期才能发布（>50% 工作量） |
| **5** | 需要中等程度人工后期（20-50% 工作量） |
| **7** | 只需少量人工微调（<20% 工作量）即可发布 |
| **10** | 全自动产出即可直接上平台分发，无需任何人工后期 |

### 基准参考：《西游玉帝》 — **8 分**

表现：
- 成品直接在红果短剧平台上线并成为爆款
- 211 模式（20 天、10 人团队），虽然不是全自动但效率已经很高
- 10 万元算力成本 + 人工成本，总成本远低于传统动画
- 仍需要人工参与资产调整、画面精修、配音后期等环节，不是一键出片

## 综合评分计算

### 加权总分公式

```
综合评分 = 角色一致性 × 0.20 + 画面质量 × 0.20 + 动态运镜 × 0.15 + 叙事节奏 × 0.20
         + 音频质量 × 0.10 + 技术完成度 × 0.10 + 商业可用性 × 0.05
```

### 等级划分

| 等级 | 综合评分 | 含义 |
|------|---------|------|
| **S** | 8.5 - 10 | 标杆级，商业爆款潜力 |
| **A** | 7.0 - 8.4 | 优秀，可商业发行 |
| **B** | 5.5 - 6.9 | 合格，能上平台但竞争力弱 |
| **C** | 4.0 - 5.4 | 勉强，需大量人工补救 |
| **D** | 0 - 3.9 | 不可用 |

### 基准作品综合评分

| 维度 | 权重 | 评分 | 加权得分 |
|------|------|------|---------|
| 角色视觉一致性 | 20% | 8.0 | 1.60 |
| 画面质量与美术风格 | 20% | 8.5 | 1.70 |
| 动态与运镜 | 15% | 7.5 | 1.125 |
| 叙事结构与节奏 | 20% | 9.0 | 1.80 |
| 音频质量 | 10% | 7.0 | 0.70 |
| 技术完成度 | 10% | 7.5 | 0.75 |
| 商业可用性 | 5% | 8.0 | 0.40 |
| **综合评分** | | | **8.075** |

**《西游玉帝》综合等级：A**（接近 S 级，主要被音频和动态拉低）

### 评分说明

- **FreeToon** 的高分主要来自"内容一线长出的工具"这一定位 — 工具本身融入了爆款制作经验（40% 爆款率远高于行业 0.16%）。全集思维、多状态资产库、AI 配音集成等能力让它在端到端交付上有明显优势。
- **巨日禄** 在融图技术上成熟，是行业最早的标杆工具，但定位为"纯工具"（不做剧本、不做配音），在叙事和音频维度上天然缺失。画面质量受"巨日禄风"争议影响，但其爆款案例证明商业上是可行的。
- **火山剧创** 架构设计先进（多 Agent 协同），内置爆款策略库是亮点，但刚邀测无法实际评估。字节生态闭环（Seedance 2.0 + 剪映 + 红果）可能在商业可用性上后期提升。
- **LibTV** 的调研价值不在漫剧维度的分数，而在架构设计：双入口（Creator + Agent）、Skill 中间层、Agent 状态管理、工作流模板作为可复用审美资产。作为通用平台在商业广告、产品 TVC 等场景的表现可能显著高于此处漫剧场景评分。详细调研见 [tools/libtv.md](../tools/libtv.md)。
- **小云雀** 依托 Seedance 2.0（当前公认最强视频模型）在画面和动态维度有天然优势，故事理解和全局角色管理使其在一致性和叙事维度表现突出。原生音画一体是音频维度的差异化亮点。免费 + C 端零门槛 + 字节分发闭环使其商业可用性高。但封闭平台、单一模型绑定、缺乏策略库是限制因素。详细调研见 [tools/xiaoyunque.md](../tools/xiaoyunque.md)。

## 自建系统质量目标

### 阶段目标

| 阶段 | 目标综合评分 | 等级 | 核心达标维度 |
|------|------------|------|------------|
| **MVP** | ≥ 5.5 | B | 角色一致性 ≥ 5，画面质量 ≥ 6，技术完成度 ≥ 5 |
| **V1** | ≥ 7.0 | A | 全部维度 ≥ 6，叙事节奏 ≥ 7（集成策略库） |
| **V2** | ≥ 8.0 | A（近 S） | 追平《西游玉帝》基准，音频和动态达 7+ |

### 各阶段重点攻关

**MVP 阶段：解决"能用"问题**
- 角色一致性：实现基础的角色资产管理 + 参考图注入
- 画面质量：接入最优模型（模型路由基础版）
- 技术完成度：分辨率达标、基础字幕生成

**V1 阶段：解决"好用"问题**
- 叙事节奏：集成爆款策略库（参考火山剧创的 200+ 策略思路）
- 音频质量：集成 TTS API（Fish Audio / ChatTTS）
- 动态运镜：优化视频模型选择策略，按镜头类型路由
- 商业可用性：降低人工后期比例至 20% 以下

**V2 阶段：解决"超越"问题**
- Agent 自动评估 + 重试（参见 [architecture-ideas.md](architecture-ideas.md) 第 7 节）
- 基于本评分体系的自动质量检测，低于阈值的镜头自动重新生成
- 全链路闭环：剧本生成 → 分镜 → 画面 → 视频 → 配音 → 字幕 → 发布

### 自动化质量评估的设想

V2 阶段的核心能力是让 Agent 能自动评估产出质量并决策是否重试：

```
生成结果 → 质量评估 Agent {
  角色一致性检测（对比参考图，计算相似度）
  画面质量评分（调用视觉模型打分）
  技术完成度检查（检测 AI 伪影/崩坏）
} → 评分 ≥ 阈值？ → 通过 / 重新生成
```

这是当前所有调研工具都未实现的能力（参见 [gaps.md](gaps.md) 第 6 节"内容评估/爆款预测"），也是自建系统最大的差异化方向之一。
