# 市场空白与机会

现有工具做不好或没人做的地方。

## 来自巨日禄调研的发现

### 1. 剧本/故事生成 — 完全空白

巨日禄不做剧本，需要用户外部提供。但杰夫自己说"好故事永远是最重要的"、"AI精品化路径就是打磨剧本"。这是一个明确的缺失环节 — 如果 Agent 能自动生成高质量剧本，就补上了链条的第一环。

### 2. 语音/配音 — 完全依赖第三方

巨日禄不做语音合成和配音，需要第三方配音团队。自建系统如果集成 TTS（如 Fish Audio、ChatTTS 等），就能实现全链路闭环。

### 3. 口型同步 — 没人做好

巨日禄不支持口型同步。这在漫剧场景里影响没那么大（静态嘴型），但在真人 AI 剧场景下是硬需求。2026 年真人剧爆发会把这个需求推上来。

### 4. "一键成片" — 竞品已在尝试

小云雀短剧 Agent（2026.03）已经在打"一键成片"概念。巨日禄目前仍需逐镜头调试，效率瓶颈在人工干预环节。Agent 驱动的自动化程度越高，越有竞争力。

### 5. API / 可编程性 — 封闭平台的软肋

巨日禄是完全封闭的平台，无 API，不可集成到自动化流水线。对于想构建自定义工作流、做二次开发的团队来说，这是硬伤。开放平台 / Agent 可编程是一个差异化方向。

### 6. 内容评估 / 爆款预测 — 事后才知道

巨日禄能帮做剧，但不能帮判断"这个剧能不能爆"。行业爆款率只有 0.16%，90% 的公司亏损。如果系统能集成内容评估/爆款预测能力（基于历史数据分析），就是巨大的差异化。

### 7. 多语言 / 出海本地化

杰夫提到"全球短剧 TOP10 APP 都来自中国"，出海是明确趋势。但巨日禄本身没有做多语言翻译、本地化适配。集成翻译 + 多语言配音 + 字幕本地化是出海场景的刚需。
