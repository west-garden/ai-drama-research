## 核心能力

### 1. 故事理解 — "看懂故事"的 Agent

小云雀短剧 Agent 不只是把剧本拆成分镜，它能**深度解析完整剧本**：

- 精准梳理世界观、时间线与人物关系
- 理解叙事氛围、每场戏的情感基调
- 自动生成完整的"故事蓝图"（story blueprint）

一个角色在第一集懦弱、第三幕果决，行为变了，但 Agent 始终知道那是同一个人——说话方式、面对某类处境时的惯性反应、和特定角色之间的相处模式，这些细节它都记得，也都对得上。

### 2. 全局角色管理 — "超级选角导演 + 服化道总监"

角色一致性是 AI 漫剧最核心的技术挑战，小云雀的方案是**全局角色生命周期管理**：

- **身份连续性**：当剧本中出现不同称谓时，Agent 不会将其视为不同角色，能通过上下文分析自动关联到同一核心角色实体
- **形象变化追踪**：自动扫描全剧本，智能分析角色全生命周期的形象变化（年龄、服饰、面部特征）
- **资产库 + 人物小传**：为每个角色维护完整的"人物小传"，确保在数十集剧情中核心身份与演变逻辑高度一致
- **可编辑**：如果对系统生成的角色图片不满意，可以替换，后续所有镜头基于替换后的形象

### 3. Seedance 2.0 底层能力

Seedance 2.0 是字节跳动自研的视频生成模型，宇树科技创始人王兴兴称其为"全球目前最好的"视频生成模型。

单论 Seedance 2.0 模型本身：最长只能生成 15 秒视频，每次需要重新贴参考、重新写提示词。小云雀 Agent 的工程化价值在于：Agent 层自动处理参考人物、分镜表、剪辑，用户只需给一个剧本，Agent 就能全自动跑出完整短剧。

Seedance 2.0 + 小云雀工程化 = 1 + 1 > 2

### 4. 原生音画一体

配音环节的逻辑是**原生音画一体**——画面与声音在同一时间维度上生成，而不是先做画面再贴配音。

- 声音从画面里"长出来"，不是贴上去的
- 整体一致性更好
- 支持自动改编旁白（将演绎类剧本转化为解说剧本）

### 5. 并行化批量生成

支持真正意义上的**并行化、非线性**视频批量生成：

- 完成剧本研读后，Agent 已为整部剧集构建完整的"数字资产档案"与"创作蓝图"
- 可以同时生成多集多镜头，不需要线性等待
- 多集连载只需完成一次剧本解析，即可逐集生成，不用每集重复上传

### 6. 多画风支持

提供三大类视觉风格：

- **2D** — 含国风半厚涂等子风格
- **3D** — 3D 渲染风格
- **仿真人** — 接近真人短剧效果

系统根据故事类型智能推荐五种画风，也支持自定义输入。

## 工作流

```
1. 上传剧本（最多 10 万字 .docx）或 AI 生成剧本（一句话创意）
        ↓
2. 确认基础设定（画风、画幅、旁白模式）
        ↓
3. Agent 自动构建（等待约 30 分钟+）
   - 剧本研读 → 故事蓝图
   - 全局角色设计 → 人物小传 + 资产库
   - 分镜生成（含场景、布景、光影、运镜、表演细节）
        ↓
4. 人工审阅/编辑（可选）
   - 查看故事蓝图、角色资产、分镜脚本
   - 逐分镜修改提示词和角色引用
   - 替换不满意的角色图片
        ↓
5. 一键生成成片 / 逐镜头生成
   - 并行化批量视频生成
   - 原生音画一体（配音同步生成）
        ↓
6. 局部重新生成不满意的镜头
```

### 分镜生成细节

Agent 生成的分镜不是简单的画面描述，包含多维度信息：

- **场景构建**：场景样貌、布景细节、光影
- **画面设计**：场景过渡、道具位置
- **表演细节**：人物站位、服饰、面部朝向、动作、表情、眼神、氛围、音色
- **运镜**：单镜头运镜时长、多镜头间的转场方式

所有要素均可人工调整。

## 对自建系统的启发

### 1. "看懂故事"是 Agent 层真正需要解决的问题

小云雀最大的突破不在画质（那是 Seedance 2.0 的功劳），而在**故事理解**。Agent 能分析角色在不同情境下的行为一致性、理解叙事弧的情感变化，这才是让"一键成片"真正可用的关键。

自建系统应把"故事理解"作为 Agent 层的核心能力：
- 剧本解析不只是文本分割，要理解世界观、人物关系、情感弧
- 角色管理不只是图像一致性，要追踪角色全生命周期的行为逻辑

### 2. 全局角色生命周期管理

小云雀的角色管理方案与巨日禄的融图、FreeToon 的资产继承不同——它是**语义层面的全局管理**：

| 方案 | 巨日禄（融图） | FreeToon（资产继承） | 小云雀（全局角色管理） |
|------|--------------|-------------------|-------------------|
| 层面 | 图像合成 | 语义追踪 | 语义 + 视觉 |
| 输入 | 参考图 + 场景 + 姿态 | 全集剧本 + 资产提取 | 全集剧本 + 角色生命周期分析 |
| 特点 | 每次生成注入参考图 | 追踪角色出场状态变化 | 理解身份连续性，不同称谓关联同一角色 |

自建系统可以综合三者：融图保证视觉层面像素级一致，资产继承保证状态变化可追踪，全局管理保证行为逻辑一致。

### 3. 原生音画一体 > 先画面后配音

小云雀的"声音从画面里长出来"的方案，比"先生成视频再贴配音"的方案在一致性上更好。自建系统在集成 TTS 时应考虑：
- 能否在视频生成阶段就注入语音信号
- 口型同步应该是生成阶段的约束，而不是后处理

### 4. 并行化非线性生成是效率关键

小云雀在完成剧本解析后，为整部剧构建"数字资产档案"和"创作蓝图"，然后可以**并行生成**多集多镜头。

自建系统的工作流引擎应该支持：
- 前期串行（剧本解析 → 资产构建 → 蓝图生成）
- 后期并行（基于蓝图，多集多镜头同时生成）
- 依赖管理（同一角色的关键镜头要确保一致性约束传递）

### 5. 免费 + 字节生态 = 对自建系统的竞争压力

小云雀免费 + 字节分发闭环（抖音 + 红果 + 剪映），对 B 端工具和自建系统形成极大竞争压力。自建系统的差异化必须来自：
- 多模型路由灵活性（不绑定单一模型）
- 可编程/可集成（API + Skill 接口）
- 不绑定特定分发平台
- 定制化能力（自定义策略库、工作流模板）
