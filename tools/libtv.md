## 产品定位

LibTV 与本项目已调研的巨日禄、FreeToon、火山剧创有本质区别：

| 维度 | 巨日禄/FreeToon/火山剧创 | LibTV |
|------|--------------------------|-------|
| 定位 | 漫剧/短剧垂直工具 | 通用 AI 视频创作平台 |
| 核心场景 | 短剧/漫剧批量生产 | 商业广告、专业影视、动漫游戏、教育生活 |
| 用户类型 | B 端制作团队 | 专业创作者 + AI Agent |
| 差异化 | 行业 know-how（融图、爆款策略等） | 双入口架构 + 多模型路由 + 低价格策略 |

LibTV 的调研价值不在于它能否做漫剧（它能，但不是专精），而在于它的**架构设计思路**——如何从产品设计第一天就把人和 Agent 当成平等的两类用户。

## 核心能力

### 1. 无限画布 + 节点工作流

LibTV 的创作主界面是一块可以无限延伸的画布，支持五种基础节点：

- **文本节点** — 剧本、人物设定、prompt
- **图片节点** — 参考图上传或 AI 生成
- **视频节点** — 视频素材或 AI 生成
- **音频节点** — 配乐、音效
- **脚本节点** — 自动生成分镜脚本

节点之间可自由连线，上游输出直接成为下游输入。整套流程可打组保存为**工作流模板**，一键复用。

### 2. 20+ 专业创作工具

包含多个行业首创功能：

| 功能 | 说明 |
|------|------|
| 9/25 宫格分镜 | 一次生成 9 或 25 张连贯分镜构图方案 |
| 剧情推演四宫格 | 根据单帧画面推演 4 格剧情发展 |
| 角色三视图 | 锁定角色形象，保持后续生成一致性 |
| 多机位镜头设计 | 多视角拍摄方案 |
| 画面延展 | 前 3 秒 / 后 5 秒画面推演 |
| 三维坐标球打光 | 专业级 3D 光照控制 |
| 15 环绕视角 | 360 度角度呈现 |
| 高清扩图 | 2x / 4x / 6x 超分辨率 |
| 视频提帧 | 提升至 90fps |
| 局部重绘/擦除/抠图 | 精细化编辑 |
| 一键故事板 | 脚本自动转分镜 |
| 摄像机控制 | 相机类型、镜头焦距、光圈调节 |

### 3. 多模型集成（30+ 模型）

LibTV 不绑定单一模型，集成了当前市面主流的 30+ AI 模型：

**视频模型：** 可灵 O3、可灵 3.0、Seedance 2.0、Pixverse V5.5、Wan 2.6、Vidu Q3 PRO、Seedance 1.5、Hailuo 2.3、Video 3.1 等

**图像模型：** Nano Pro、Seedream 5.0、Midjourney V7、Qwen Image 等

**音频模型：** Mureka 音频生成

### 4. Agent 双入口架构

这是 LibTV 最核心的架构创新——从产品设计第一天起，人类和 Agent 就有各自独立的入口。

**创作者入口（GUI 画布）：**
- 无限画布 + 节点工作流
- 20+ 专业工具的图形化操作
- 工作流模板保存和复用

**Agent 入口（libtv-skills）：**
- 开源 Skill 包（MIT 协议）
- 支持 OpenClaw、KimiClaw 等 Agent 直接调用
- Agent 用自然语言表达意图，系统侧完成编排
- 安装：`npx skills add libtv-labs/libtv-skills`
- 认证：`LIBTV_ACCESS_KEY` 环境变量
- API 基地址：`https://im.liblib.tv`

**Agent 返回三要素：**

| 返回值 | 用途 |
|--------|------|
| `sessionId` | Agent 持续查询进展 |
| `projectUuid` | 创作挂在可持续的项目对象上 |
| `projectUrl` | 人类随时打开画布接管 |

Agent 交付的不是一个孤立的 mp4 文件，而是一整个可以继续编辑的画布项目。人类打开 projectUrl 就能接着改。

### 5. 批量生成与自动比选

Agent 没有人类的直觉审美，它的办法是靠量：每个镜头生成多个版本，按一致性、构图、风格匹配度自动筛选。用算力换审美。

这也解释了为什么 LibTV 的低价策略至关重要——Agent 天然需要多版本生成和比选，调用频次比人手动操作高得多。

## 工作流

### 创作者端流程

```
1. 打开画布 → 放置基础节点（文本/图片/视频/音频/脚本）
        ↓
2. 节点内生成内容（选模型、调参数、生成）
        ↓
3. 节点间连线（上游输出 → 下游输入）
        ↓
4. 使用专业工具精修（三视图、多机位、打光、扩图等）
        ↓
5. 打组保存为工作流模板 → 添加到工具箱 → 下次复用
```

### Agent 端流程

```
1. 用户给 Agent 一句话指令（如"做一个 2 分钟的赛博朋克动画短片"）
        ↓
2. Agent 调用 LibTV Skill → 自动拆解剧本、设计角色、生成分镜
        ↓
3. LibTV 返回 sessionId + projectUuid + projectUrl
        ↓
4. Agent 逐镜头生成视频、配乐、剪辑
        ↓
5. 交付成片链接 + 可编辑画布项目
        ↓
6. 人类可随时打开 projectUrl 接管编辑
```

### 脚本生成三模式

| 模式 | 说明 |
|------|------|
| 剧本生成分镜脚本 | 输入文字剧本，自动拆解为分镜 |
| 视频参考生成分镜脚本 | 输入参考视频，AI 分析后生成对应脚本 |
| 角色生成分镜脚本 | 基于角色设定生成故事和分镜 |

## AI 能力边界

- **能自动完成的：** 剧本生成、分镜拆解、角色设计、多模型画面/视频生成、配乐、批量生成与比选、工作流编排、视频剪辑

## 技术架构亮点

### 双轨架构设计哲学

LibTV 不是"先做 GUI 再补 API"，而是从产品架构层面，人类和 Agent 的入口就是并行设计的。

传统模式：
```
GUI 产品 → 后补 API → Agent 通过 API 模拟人类操作
```

LibTV 模式：
```
共享能力层（模型、工作流、项目管理）
    ├── 创作者入口：GUI 画布（针对人类认知优化）
    └── Agent 入口：Skill 接口（针对 Agent 执行优化）
```

### Skill 中间层：不是 API 也不是一键出片

a16z 和红杉分析 Agent 产品设计时都提到了「Goldilocks」原则——最优解在中间层。

| 层级 | 特点 | 问题 |
|------|------|------|
| 碎片化 API | 一次调一个功能 | Agent 得来回调几十次，效率低 |
| 一键出片 | 黑盒全自动 | 不同任务没法差异化，无法介入 |
| **Skill（中间层）** | **意图到编排** | **Agent 表达意图，系统完成编排** |

LibTV 的 Skill 不是把 GUI 操作翻译成代码调用，而是让 Agent 用自然语言表达意图，由系统侧完成编排和决策。每个 Skill 里有完整的决策链路，但 Agent 可以在不同 Skill 之间自由组合。

### Agent 状态管理

Agent 没有人类的「空间记忆」（看一眼画布就知道整体状态），需要系统帮它管理状态：

```
sessionId    → Agent 的"对话记忆"，持续查询创作进展
projectUuid  → 项目级状态，角色约束、前面镜头的色调等
projectUrl   → 人机切换的桥梁，随时让人接管
```

### 工作流模板 = 可流通的审美资产

创作者在画布上调好的工作流存成模板，模板记录的不只是"用了哪些节点、连了哪些线"，还有每个环节的参数偏好：镜头时长、构图倾向、色调范围、节奏结构。

这意味着审美判断可以从个人直觉变成**可存储、可流通、可被 Agent 调用**的资产。

## 优点

- **双入口架构** — 从产品设计第一天就同时为人和 Agent 设计，不是"先做 GUI 再补 API"的后补逻辑
- **多模型路由** — 30+ 模型集成，不绑定单一供应商，创作者可按需选择最优模型
- **价格策略激进** — 比 Runway/Pika 低 76-92%，为 Agent 批量生成提供经济基础
- **开源 Skill** — MIT 协议，支持 OpenClaw/KimiClaw 等 Agent 生态
- **Agent 交付可编辑项目** — 不是 mp4 死文件，而是可继续编辑的画布项目
- **工作流模板可复用** — 审美判断资产化，可在社区流通
- **20+ 专业功能** — 9/25 宫格分镜、角色三视图、多机位等，部分行业首创
- **创作者生态** — 背靠 2000 万创作者社区和 50 万+原创模型

## 缺点

- **非漫剧垂直** — 通用平台在漫剧场景缺乏开箱即用的流水线，需自行搭建工作流
- **无爆款策略库** — 不像火山剧创内置 200+ 策略，缺乏短剧叙事方法论
- **角色一致性有限** — 有三视图但非融图/资产继承级别，跨镜头仍有偏差
- **配音能力弱** — TTS 配音、口型同步未重点集成
- **平台较新** — 2026 年 3 月上线，功能仍在早期打磨阶段
- **无多集连续性** — 缺乏全集思维的数据结构和资产追踪
- **需登录才能创作** — 画布功能需注册账号
- **字幕支持未确认** — 多语言字幕能力未在公开信息中提及

## 对自建系统的启发

### 1. Agent 入口的产品设计：双入口 > 后补 API

LibTV 最重要的启发是**从产品设计第一天就把 Agent 当成一类真实用户**，而不是做完 GUI 后再补一个 API。

具体到自建系统：
- 能力层应该是共享的（模型调用、工作流引擎、项目管理）
- 人类和 Agent 各自有优化过的入口
- Agent 入口不是 GUI 操作的翻译，而是基于意图的编排

### 2. Skill 中间层：意图到编排的抽象

碎片化 API 太细（Agent 需要来回调几十次），一键出片太粗（不同任务没法差异化）。Skill 是中间层——Agent 用自然语言表达意图，系统侧完成编排。

自建系统可以参考：
- 不要只暴露底层 API，要封装"创作意图"级别的 Skill
- 每个 Skill 内有完整决策链路，但 Skill 之间可自由组合
- 遵循 OpenClaw Skill 规范，可被主流 Agent 直接调用

### 3. Agent 状态管理：sessionId / projectUuid / projectUrl

Agent 每次创作返回三要素的设计值得借鉴：
- `sessionId` 保持对话连续性
- `projectUuid` 保持项目级状态
- `projectUrl` 实现人机无缝切换

自建系统需要类似的状态管理机制，让 Agent 交付的不是死文件，而是可继续编辑的项目。

### 4. 多模型路由 + 低价格策略

LibTV 集成 30+ 模型的策略验证了"不绑定单一模型"的正确性。配合激进的价格策略（比竞品低 76-92%），为 Agent 的批量生成提供了经济基础。

自建系统需要考虑：
- 模型路由层是核心中间件
- Agent 天然需要多版本生成 + 自动比选，单价必须低
- 与模型厂商的长期合作关系决定成本结构

### 5. 工作流模板作为可复用审美资产

创作者调好的工作流模板不只是"技术参数"，还带着审美判断。这些模板可以在社区流通，Agent 调用时也会带着前一个创作者的审美印记。

自建系统可以参考：
- 分镜策略库（参考火山剧创的 200+ 策略）可以设计成"可流通的模板"
- 模板 = 节点 + 连线 + 参数偏好 + 审美判断
- 让"审美"从锁在个人直觉变成可编程、可复制的资产

### 6. Agent 创作的经济学

Agent 天然是"多调几倍模型"的用户——批量生成 + 自动比选需要低单价支撑。如果单次调用太贵，Agent 做视频在经济上不成立。

自建系统的成本模型必须考虑：
- Agent 的调用频次是人类的 3-10 倍
- 模型路由不只是选最好的，还要选性价比最高的
- 生成失败/不满意的重试成本要纳入预算
